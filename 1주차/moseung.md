# 프레임워크에 대한 이야기
### 이 장에서 설명하는것
- 자바스크립트 프레임워크에 대한 전반적인 이야기

## 프레임워크란?
**무언가를 만들 수 있는 지지구조**
<br>
lodash나 momentJs와 같은것은 라이브러리라 부르고 라이브러리와 프레임워크의 가장 큰 차이는 프레임워크는 코드를 호출하고 코드는 라이브러리를 호출한다는것이다.

### 언어
Typescript가 현재 웹개발의 사실상 표준이다. Typescript는 일반 자바스크립트로 컴파일되는 자바스크립트의 형식화된 슈퍼 세트이다.(정적 타입분석) 하지만 앵귤러를 사용하면 작성되는 모든 코드는 트랜스파일러가 필요하다.

### 의존성 주입
요소가 앵귤러와 통신할 수 있게 하려면 유형에 따라 의존성 메커니즘을 사용해 요소를 주입해야 한다.

### 옵저버블
앵귤러는 옵저버블을 사용한 반응형 프로그래밍용 라이브러리인 RXJs를 기반으로 설계됐다. 
[RXJs를 잘 소개한 글](https://velog.io/@teo/rxjs)
<br>

### 리액트에 대해 이야기해보자
```
React는 JavaScript 라이브러리로 분류됩니다. React는 사용자 인터페이스를 구축하기 위한 선언적인 방식을 제공하며, 이를 통해 복잡한 UI를 개발할 때 상태 관리와 컴포넌트 간의 상호작용을 쉽게 할 수 있습니다.

그러나 React 자체는 전체 웹 애플리케이션을 구축하는 데 필요한 모든 기능을 제공하지 않습니다.
<br>
예를 들어, HTTP 요청 처리, 라우팅, 상태 관리 등은 React 자체에서 지원하지 않으며 이런 부분들은 추가적인 라이브러리나 도구(예: Axios, React Router, Redux 등)의 도움을 받아야 합니다.

반면에 프레임워크는 전체적인 솔루션을 제공하며, 개발자가 프레임워크 내에서 규정된 규칙과 패턴에 따라 코드를 작성하도록 합니다. Angular와 Vue.js가 이런 종류의 프레임워크에 해당합니다.

따라서 엄밀히 말해서 React는 "라이브러리"입니다. 하지만 다른 도구들과 결합하여 사용될 때 전체적으로 프레임워크처럼 작동할 수 있습니다.
```
리액트의 주요 제약 사항은 선언적 패러다임의 사용이다. DOM을 직접 조작하는것이 아니라 상태를 조작한다.

### 자바스크립트 프레임워크 연혁

#### 제이쿼리
선택자 구문, AJAX 요청, 애니메이션, 기타 유틸리티 같은 많은 기능이 추가된 프론트엔드 개발의 맥가이버 칼.
<br>
쉽게 플러그인할 수 있는 jQueryUI라는 공식 UIKit을 갖고 있어 웹에서 모든 요구사항을 만족시킬 수 있게 됐다.

#### 앵귤러JS
제이쿼리가 문자의 발견이라면 앵귤러JS는 쿠텐베르크의 인쇄술.
<br>
SPA가 주류로 만드는데 큰 역할을 했고 가장 주목할만한 기능은 양방향 데이터 바인딩이다.
<img width="528" alt="image" src="https://github.com/FrontendStudySeoul/FrontEndDevelopNoFramework/assets/103626175/090175fa-e9c4-400c-aace-e0988f1c297b">

#### 리액트
현재 가장 인기있는 프론트엔드 라이브러리(프레임워크).
<br>
리액트는 선언적 패러다임으로 동작한다. 일반적으로 DOM을 직접 수정하는 대신 setState로 상태를 변경하면 리액트가 나머지 작업을 수행한다.
<br>
기술적으로 본다면 리액트는 프레임워크가 아니라 렌더링 라이브러리이다.

#### 앵귤러
앵귤러는 앵귤러JS의 새로운 버젼으로 시작됐다. 엔터프라이즈 세계를 타깃으로 잡았따.

#### 프레임워크 비용
모든 프레임워크가 기술 부채를 갖고 있다.<br>
그리고 프레임워크는 아키텍처 자체에 이미 비용을 포함하고 있다.
<br>
FEConf 연사분의 예시에 따르면 리액트를 사용할떄 리액트의 방식때문에 html의 특정 요소가 지워진다. 이는 프레임워크의 기술적 부채이며 이를 해결하기 위해 다른 노력을 기울여야한다.

#### 기술투자
기술 부채가 항상 나쁜것은 아니다.
<br>
합리적인 이유로 빠른 솔루션을 사용한다면 그것은 기술 부채가 아니라 기술 투자가 된다. 합당한 이유로 선정된 프레임워크는 비용이 아니라 자산이다.
